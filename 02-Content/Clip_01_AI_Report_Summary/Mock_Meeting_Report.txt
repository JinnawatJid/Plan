# รายงานการประชุม: โครงการปรับปรุงระบบฐานข้อมูลและคลาวด์ (Q3/2024)
**วันที่:** 15 กันยายน 2024
**เวลา:** 09:00 - 11:30 น.
**ผู้เข้าร่วมประชุม:** ทีมบริหาร, ทีม IT Infrastructure, ทีมพัฒนาซอฟต์แวร์, ทีม Security
**ผู้บันทึกการประชุม:** สมชาย ใจดี

---

## 1. วาระที่ 1: การอัปเดตความคืบหน้าการย้ายฐานข้อมูล (Database Migration)
คุณวิชัย (Lead IT) รายงานว่าการย้ายฐานข้อมูลลูกค้าจากระบบ On-Premise ไปยัง Cloud (AWS) มีความล่าช้ากว่ากำหนดการเดิม 2 สัปดาห์ สาเหตุหลักมาจากพบปัญหาความเข้ากันได้ของโครงสร้างข้อมูล (Data Schema Incompatibility) ระหว่างระบบเก่ากับระบบใหม่ ทำให้ต้องมีการเขียนสคริปต์เพื่อทำ Data Mapping ใหม่ทั้งหมด ซึ่งคาดว่าจะใช้เวลาเพิ่มอีกประมาณ 7 วันทำงาน นอกจากนี้ทีมงานยังมีข้อกังวลเรื่องของ Downtime ที่อาจจะเกิดขึ้นในช่วงที่สลับระบบ ซึ่งอาจกระทบต่อการใช้งานของลูกค้าในวันเสาร์-อาทิตย์หน้า

ทางคุณสมศักดิ์ (CEO) ได้สอบถามถึงแผนสำรอง (Rollback Plan) ในกรณีที่การสลับระบบล้มเหลว ทางคุณวิชัยชี้แจงว่าได้เตรียม Snapshot ของฐานข้อมูลเดิมไว้แล้ว หากมีปัญหาจะสามารถกู้คืนได้ภายใน 2 ชั่วโมง แต่จะต้องสูญเสีย Transaction ที่เกิดขึ้นในช่วงเวลาดังกล่าว ซึ่งทีมงานกำลังหาวิธีลดความเสี่ยงตรงนี้อยู่

## 2. วาระที่ 2: ปัญหาคอขวดในระบบ Network (Network Bottleneck Issues)
คุณมาลี (Network Engineer) นำเสนอข้อมูลจากระบบ Monitoring พบว่าในช่วงเวลา Peak Hours (10:00-11:30 น. และ 14:00-15:30 น.) มีการใช้งาน Bandwidth สูงถึง 95% ของความจุทั้งหมด ทำให้แอปพลิเคชันภายใน (Internal ERP) ตอบสนองช้าลงอย่างเห็นได้ชัด (Latency เพิ่มขึ้นจาก 50ms เป็น 300ms)
ปัญหาหลักเกิดจากการดึง Report ขนาดใหญ่ของทีม Data Analytics ที่รันในช่วงเวลาเดียวกัน คุณมาลีเสนอให้ปรับตารางการรัน Report ของทีม Analytics ไปเป็นช่วงเวลากลางคืนแทน (Batch Processing)

คุณนที (Data Team Lead) แจ้งว่าสามารถปรับเวลาได้ แต่ต้องขออนุมัติจากฝั่ง Business ก่อน เนื่องจากผู้บริหารบางท่านต้องการดู Dashboard แบบ Real-time ในตอนเช้า ทางทีมจึงขอให้มีการขยาย Bandwidth หรือเพิ่มการทำ Caching Server

## 3. วาระที่ 3: การประเมินความปลอดภัยทางไซเบอร์ (Security Audit Findings)
คุณเอกราช (Security Officer) รายงานผลการทดสอบการเจาะระบบ (Penetration Testing) ประจำไตรมาส พบช่องโหว่ระดับ High จำนวน 3 จุดในระบบ API ที่เปิดให้พาร์ทเนอร์ใช้งาน (Partner Portal) ช่องโหว่ดังกล่าวอาจทำให้ผู้ไม่หวังดีสามารถเข้าถึงข้อมูลบางส่วนข้ามสิทธิ์ได้ (Insecure Direct Object Reference - IDOR)
ทีม Security แนะนำให้ทีมพัฒนาซอฟต์แวร์หยุดการปล่อยฟีเจอร์ใหม่ชั่วคราว (Feature Freeze) และมุ่งแก้ไขช่องโหว่นี้ให้เสร็จสิ้นภายในสัปดาห์นี้

นอกจากนี้ ยังพบว่าพนักงานกว่า 30% ยังไม่ได้ทำการอบรม Security Awareness ประจำปี ซึ่งเกินกำหนดเวลามาแล้ว 1 เดือน ขอให้ HR ช่วยติดตามอย่างเร่งด่วน

## 4. วาระที่ 4: การจัดซื้อ Software License เพิ่มเติม (Software License Procurement)
ทีมงาน Development แจ้งว่า License ของเครื่องมือ CI/CD ที่ใช้อยู่ปัจจุบัน (GitLab Premium) กำลังจะหมดอายุในอีก 45 วัน และเนื่องจากทีมมีขนาดใหญ่ขึ้น ทำให้ต้องซื้อ User License เพิ่มอีก 15 Seats
คุณวิชัยเสนอให้พิจารณาเปรียบเทียบราคากับการย้ายไปใช้ GitHub Enterprise ซึ่งอาจจะคุ้มค่ากว่าในระยะยาว ทางทีมจัดซื้อรับเรื่องไปเพื่อขอใบเสนอราคา (Quotation) จาก Vendor ทั้งสองเจ้า และจะนำมาสรุปให้ผู้บริหารตัดสินใจในวันพุธหน้า
